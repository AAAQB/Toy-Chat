## 一、产品技术整体架构

### 1.1 产品形态

**Brighten** 是一款面向 6-12 岁儿童的「AR + AI 对话」教育互动平台，以 Web 应用为载体，用户通过手机摄像头扫描实体玩具人偶，即可唤醒对应职业角色并与之进行自由对话。

| 维度 | 技术选型 |
|---|---|
| 前端架构 | 多页 HTML（SPA 混合）+ 原生 Vanilla JavaScript |
| 对话引擎 | DeepSeek Chat API（deepseek-chat 模型），SSE 流式传输 |
| AR 引擎 | A-Frame v1.4 + MindAR v1.2.2（图像追踪，WebAssembly + WebGL） |
| 音效系统 | Web Audio API，自研多层合成波形管线 |
| 粒子特效 | Canvas 2D + requestAnimationFrame（重力模拟 + 辉光渲染） |
| 多语言 | 自研 i18n 运行时引擎，支持 en / zh / ms 三种语言热切换 |
| 数据持久化 | localStorage（仅存储语言偏好） |
| 部署方案 | 纯静态资源托管，CDN 加速，零服务端运维 |

### 1.2 系统架构分层

系统采用自顶向下的五层架构设计，各层之间通过标准接口进行通信：

```
第一层：前端展示层 (Presentation Layer)
  ├─ 页面：Home / AR Scan / Career Detail / Careers Grid
  ├─ 职责：页面渲染、用户交互、页面路由
  └─ 技术栈：HTML5 + CSS3 + ES2020+

第二层：交互引擎层 (Interaction Engine Layer)
  ├─ 组件：MindAR 图像追踪 / DeepSeek AI 对话 / Canvas 粒子系统 / Web Audio 音效
  ├─ 职责：AR 视觉识别、AI 对话流式输出、动态特效渲染、音频合成
  └─ 技术栈：A-Frame + MindAR / fetch + SSE Stream / Canvas 2D / AudioContext

第三层：业务逻辑层 (Business Logic Layer)
  ├─ 模块：角色状态管理 / Fallback 故障切换 / 会话上下文管理
  ├─ 职责：对话生命周期控制、异常处理、上下文窗口管理
  └─ 核心机制：FIFO 上下文裁剪（上限 21 条）、AbortController 请求取消

第四层：数据层 (Data Layer)
  ├─ 组件：Locale Store / Conversation History / Preset 配置
  ├─ 职责：用户偏好存储、会话历史维护、角色人格提示词管理
  └─ 存储方案：localStorage（持久化）+ 运行时内存（对话状态）

第五层：运维部署层 (Deployment Layer)
  ├─ 策略：静态文件托管 / CDN 加速 / 浏览器缓存策略
  └─ 缓存规则：静态资源 max-age=31536000；HTML no-cache
```

## 二、平台底层逻辑

### 2.1 用户流转链路

```
用户进入主页 (Home)
│
├─ 浏览职业英雄列表 (Careers Grid)
│    └─ 查看职业详情 (Career Detail)
│         └─ 内容：3 项工作职能展示 + 3 条冷知识 + AR 入口
│
└─ AR 扫描页 (AR Scanning)
     │
     ├─ AR 扫描模式：MindAR 实时图像追踪
     │    ├─ 识别命中 -> onCareerFound() 回调 -> 底部对话面板展开
     │    ├─ 目标丢失 -> 2 秒延迟后隐藏面板
     │    └─ 设计要点：底部面板状态下 AR 摄像头保持活跃
     │
     └─ 聊天模式：顶部「💬 Chat」按钮入口
          ├─ 弹出 2x3 角色选择面板（带级联入场动画）
          ├─ 选择角色 -> 触发全屏转场动画（持续约 2.8 秒）
          └─ 转场完成后进入全屏对话界面
```

### 2.2 数据驱动机制

| 环节 | 触发机制 |
|---|---|
| AR 目标识别 | MindAR 加载 `targets.mind` 特征图库，6 个 targetIndex 对应 6 个职业角色 |
| 角色唤醒 | 识别命中后自动匹配 CAREER_PROMPTS 中的 System Prompt，注入对话引擎 |
| 对话回复 | 用户输入 -> 拼接历史上下文（上限 21 条）-> DeepSeek API -> SSE 流式解析 -> 逐 chunk 渲染 |
| 故障切换 | API 异常（网络中断 / HTTP 错误 / 请求中止）-> 自动切换至本地 Fallback 文案池 |
| 多语言切换 | data-i18n 属性绑定 + JavaScript 运行时引擎，locale 变更时自动重渲染全部文本 |
| 视觉反馈管线 | 消息发送时同步触发：音效合成 + 粒子爆发 + 打字机动画 + emoji 散落 |

### 2.3 核心技术机制

**A. SSE 流式响应管线**

单次消息发送按以下五层时序执行：

1. **音效层**：播放五层叠加合成音效（频率序列 660Hz -> 880Hz -> 1100Hz -> 1320Hz -> 1760Hz，sine + triangle 波形混合）
2. **粒子视觉层**：从发送按钮位置爆发 8 个粒子（`#FF6B6B`）
3. **等待指示层**：显示三点打字动画
4. **响应渲染层**：SSE 每 chunk 到达后以 3 字符为粒度执行打字机输出，同步播放 typing 音效并自动滚动至底部
5. **完成反馈层**：回复输出完成后触发 15 个粒子爆发 + 8 个 emoji 飞散动画

**B. 双容器对话引擎复用（DOM ID Remapping）**

同一套 `chat.js` 逻辑模块通过 `document.getElementById` 劫持机制服务于两套 UI 容器：

| 容器类型 | 适用场景 | 样式特点 |
|---|---|---|
| 底部面板 | AR 扫描模式 | 半屏高度，AR 摄像头保持可见 |
| 全屏面板 | 聊天模式 | 覆盖整个视口，沉浸式体验 |

切换时通过 `remapChatIds('fullscreen'|'panel')` 方法动态重映射 DOM ID 对照表，实现零代码修改的 UI 容器切换。

**C. 多角色人格化 System Prompt 系统**

系统维护 6 套独立角色人格定义（程序员 / 警察 / 老师 / 农民 / 医生 / 宇航员），每套 Prompt 明确定义：

- 语气风格：友好、鼓励、积极
- 语言复杂度：6-12 岁适龄词汇，回复长度控制在 6-8 句
- 表情符号使用：适度点缀，不滥用
- 知识边界：限定在角色职业领域内

所有对话内容由 DeepSeek API 实时生成，不依赖预设回复库。

**D. 会话上下文管理**

| 参数 | 配置 |
|---|---|
| 上下文窗口上限 | 21 条（1 条 system prompt + 20 轮对话） |
| 溢出策略 | FIFO 淘汰，保留首条 system prompt + 最近 20 轮 |
| 请求取消 | AbortController 确保同一时刻仅有一个活跃请求 |
| 资源清理 | beforeunload 事件触发自动回收（abort fetch、cancel animation、close AudioContext） |

**E. 音频合成引擎**

`createRichSound(layers[])` 方法支持多层波形叠加合成：

- 波形类型：sine / triangle / square
- 层参数：频率、持续时长、衰减系数、音量
- 输出处理：自动归一化至 [-1, 1] 区间，防止削波失真

**F. 粒子渲染系统**

- 渲染管线：Canvas 2D + requestAnimationFrame
- 粒子属性：位置、速度矢量、重力系数、尺寸、颜色、生命值、衰减率
- 视觉效果：shadowBlur 辉光渲染，颜色与主题匹配
- 接口设计：`(count, color, x?, y?)` 参数签名，粒子沿圆周方向均匀分布

**G. i18n 多语言运行时引擎**

- 支持语言：英语（en）、简体中文（zh）、马来语（ms）
- 词条总量：每语言约 500+ 条 key，总计约 1500+ 条
- 渲染方式：通过 `data-i18n` 属性声明式绑定，`applyTranslations()` 批量扫描替换
- 字体策略：中文环境自动加载 ZCOOL XiaoWei 衬线字体
- UI 组件：圆角地球图标下拉菜单，SVG 图标 + 语言徽章 + 级联入场动画
- 事件机制：locale 变更触发 `localechange` 自定义事件，全局联动

### 2.4 儿童内容安全与适龄设计

#### A. System Prompt 指令约束

所有角色 Prompt 在系统层面执行以下行为边界限定：

- 语言复杂度：明确限制为「简单词汇」「6-8 句中等等度」
- 情绪基调：限定为「友好」「积极」「鼓励」
- 话题范围：限定在职业知识领域，非职业话题自动引导回教育主题
- 内容禁止：系统指令明确禁止生成任何暴力、恐怖或不安全内容

#### B. 故障切换安全保护

API 异常时系统自动切换至本地 Fallback 文案池，不向用户暴露技术错误栈或 API 密钥等敏感信息。所有 Fallback 文案经人工审核编写，确保离线状态下输出内容的安全性。

#### C. 敏感话题处理策略

| 场景分类 | 处理策略 |
|---|---|
| 生命价值 / 自我伤害相关 | System Prompt 约束角色将话题引导至职业教育范畴，不展开相关讨论 |
| 暴力与伤害内容 | 角色人格设定为友好正向类型，不生成任何暴力或伤害性描写 |
| 隐私安全 | 角色行为规范不索要用户个人信息，对话范围限定于教育主题 |
| 不当言论 | i18n 界面文案经审核，不出现任何不适龄表达 |

#### D. 界面适龄性设计

- 交互模式：预设按钮 + 文本输入 + 玩具扫描，均为低门槛单步操作
- 辅助输入：不擅长打字的儿童可通过预设问题按钮直接提问
- 正向反馈：每次操作均触发音效 + 粒子 + 动画反馈
- 安全规范：无广告展示、无第三方链接、无社交功能、无内置购买弹窗
- 字体适配：中文环境采用 ZCOOL XiaoWei 衬线体优化阅读体验

#### E. 监护人侧设计

- 无后台网络请求、无第三方 Cookie 追踪
- 数据持久化范围仅限于语言偏好设置
- 对话历史存储于运行时内存，页面关闭即自动清除
- 无需注册、无需登录、无需个人信息即可使用全部功能

## 三、核心技术亮点与竞争壁垒

### 3.1 自研技术亮点

**1. 零框架原生架构**
无第三方前端框架依赖（无 React / Vue / Angular），无构建工具、无包管理器（npm）。核心脚本体积合计约 78 KB，页面首屏加载可在低速网络环境下达到亚秒级。

**2. SSE 流式响应与实时特效管线**
基于 DeepSeek API 的 SSE 流式传输，在单条回复路径上集成音效、粒子、打字机、emoji 四层同步反馈，形成完整的「发送-等待-逐字输出-完成」闭环体验。

**3. 合成音频引擎**
通过多层波形叠加（sine / triangle / square）、指数衰减与自动归一化，无需外部音频资源即可实现丰富的交互音效，为后续个性化声线定制预留了充足的扩展空间。

**4. DOM ID 重映射方案**
通过劫持 `document.getElementById` 方式实现同一对话逻辑模块在两种 UI 容器间的无缝切换，代码复用率达到 100%，显著降低了维护成本。

**5. 全自动故障切换体系**
涵盖 API 异常、网络故障、请求超时等多种异常场景，通过本地 Fallback 池 + AbortController 组合方案实现零中断切换。

**6. 轻量化 i18n 引擎**
支持运行时语言热切换，无需页面刷新。中文环境自动适配专用字体，通过 `data-locale` CSS 选择器联动字体变量。

**7. 全 CSS 转场动画系统**
独立封装的过渡动画层（`transition.css`）涵盖渐变背景、粒子闪烁、角色入场缩放与模糊消除、名称弹簧弹出、标签旋转入场、进度条动画，总持续约 2.8 秒。

## 五、产品路线图

Brighten 当前版本已完成核心 MVP 功能闭环（扫描玩具 -> AR 识别 -> AI 对话），以下为阶段性规划：

### 5.1 交互体验升级

**语音对话系统**：计划集成 Web Speech API（语音识别 + 语音合成），支持儿童直接通过语音进行交互，降低低龄用户（6-8 岁）的文字输入门槛。后续可接入 Edge TTS 或 ElevenLabs 等高质量语音合成引擎，为每个角色定制专属声线。

**互动教育游戏**：在对话流程中嵌入由 AI 角色主持的轻量级教育游戏，包括但不限于：
- 职业猜谜互动（角色描述工具或场景，用户猜测）
- 知识问答挑战（选择题形式，答对获得虚拟徽章奖励）
- 角色扮演模拟（以情景对话形式模拟职业工作流程）

所有游戏内容由 AI 实时生成，无需预设题库，保障内容的多样性和可扩展性。

### 5.2 内容生态扩展

**角色拓展**：新增更多职业角色（厨师 / 消防员 / 科学家 / 飞行员 / 艺术家等），每套角色配套独立人设 Prompt、AR 追踪模型及专属冷知识库。

**多语言覆盖**：在现有中 / 英 / 马三语基础上，逐步扩展至日语、韩语、泰语、越南语等东南亚语言，以及法语、西班牙语等全球主要语言。

### 5.3 后端基础设施建设

- 用户账号体系（邮箱注册 / OAuth 第三方登录）
- 对话记录云端同步
- 家长控制面板（对话历史查看、使用时长管理）
- 用户行为分析系统（优化 Prompt 质量与话题引导）
- A/B 测试平台（多版本 Prompt 效果对比）

### 5.4 商业化路径

- 角色套装销售：新玩具 = 新职业 IP，通过硬件绑定数字体验
- 增值订阅服务：深度职业课程、角色故事模式、自定义角色
- 教育机构合作：批量授权 + 课程大纲定制

### 5.5 工程优化方向

- PWA 化：Service Worker 离线缓存与桌面安装支持
- 客户端内容安全过滤：在 AI 输出渲染前部署关键词匹配 + 话题检测，构建双层安全机制
